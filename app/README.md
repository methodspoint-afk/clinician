# Реестр патопсихологических методик (MVP)

Приложение для клинических психологов: ввод результатов патопсихологических методик,
автоматический расчёт производных показателей, подбор валидной нормы по логике
Gate → Score и сравнение результата с нормой. Все данные хранятся локально, ФИО
испытуемых в системе не существует.

Документация проекта — в папке `../clinician-os/`:
- `technical-specification-v1-for-prompt.md` — ТЗ v1.1;
- `norm-selection-and-scoring.md` — логика отбора и оценки норм;
- `development-prompt.md` — промпт, по которому написан этот код.

## Запуск (веб-версия для разработки)

```bash
npm install
npm run dev        # открыть http://localhost:5173
```

Данные сохраняются в localStorage браузера (SQLite через sql.js).
Резервную копию можно скачать/восстановить на экране «Настройки».

## Тесты

```bash
npm test           # юнит-тесты доменного ядра и слоя БД (vitest)
npm run e2e        # сквозной браузерный сценарий (нужен Chromium;
                   # путь задаётся переменной CHROMIUM_PATH)
```

Перед e2e нужно собрать и поднять предпросмотр:

```bash
npm run build && npm run preview -- --port 4173
```

## Десктоп-сборка (Tauri 2)

Каркас лежит в `src-tauri/`. Сборка выполняется на машине с установленным Rust:

```bash
npm install
npx tauri icon путь/к/иконке.png   # сгенерировать иконки (один раз)
npx tauri build                    # установщик под текущую ОС (Windows: .msi/.exe)
npx tauri dev                      # запуск в режиме разработки
```

## Структура

```
src/domain/          чистая логика: формулы, Gate, Score, отклонения (покрыта тестами)
src/db/              SQLite: миграции, репозитории, адаптер sql.js
src/ui/              экраны на React (русский интерфейс)
e2e/acceptance.mjs   сквозной приёмочный сценарий из development-prompt.md
src-tauri/           обёртка Tauri 2 для десктопа
```
