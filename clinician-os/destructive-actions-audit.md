# Аудит разрушительных действий (июль 2026)

> Повод: в раннем MVP восстановление резервной копии молча затирало данные без
> подтверждения, проверки файла и страховки. После этого введено правило (в
> `CLAUDE.md`): любое разрушительное действие = подтверждение + проверка входа +
> откат + тест на «неправильный» путь. Этот документ — разовая проверка всех
> операций записи в приложении на соответствие правилу.

## Метод

Прошёл по всем местам, где приложение пишет/меняет данные (grep по DELETE/UPDATE/
ON CONFLICT/localStorage + разбор UI-кнопок). Классификация по риску необратимой
потери данных пользователя.

## Результат

### 🔴 Разрушительные — требовали защиты

| Операция | Где | Было | Стало |
|---|---|---|---|
| Восстановление резервной копии | SettingsScreen | молча затирало ВСЮ базу, без проверки файла и отката | ✅ проверка файла (`inspectBackup`) + подтверждение со сводкой + авто-страховка текущих данных перед заменой + тесты на чужой/битый файл |
| Импорт каталога норм (файл и сервер) | SyncScreen | молча перезаписывал локальные нормы с тем же id | ✅ теперь показывает «добавлено N, перезаписано существующих M» (`diffCatalogAgainst`), сообщение-предупреждение при перезаписи + тест |

### 🟡 Перезапись в месте — обратимо/ограниченный риск (приемлемо, под наблюдением)

| Операция | Где | Оценка |
|---|---|---|
| Редактирование нормы (в той же версии) | NormsScreen | Перезаписывает поля нормы. Смягчение: есть штатный путь «Новая версия» (старая сохраняется), доступ только у Owner. Данные пациентов не затрагиваются. **Рекомендация:** при правке уже применявшейся валидированной нормы — предлагать «новую версию» вместо тихой перезаписи. Не срочно. |
| Редактирование конфигурации методики | MethodsScreen | Перезаписывает config. Изменение (напр. удаление замера) может рассогласовать чтение прошлых результатов. Owner-only, JSON проверяется на структуру. **Рекомендация:** предупреждать, если у методики уже есть сохранённые обследования. Не срочно. |
| Веса скоринга | SettingsScreen | Перезапись настройки, полностью обратимо (кнопка «вернуть по умолчанию»), пересчёт не трогает исходные данные. Норма. |

### 🟢 Безопасные (additive / reversible / read-only)

- **Новая версия нормы** — старая версия деактивируется, но не удаляется (версионирование). Безопасно by design.
- **Смена статуса нормы** (валидировать/отклонить/архив) — обратимый статус, норма сохраняется.
- **Создание испытуемого / обследования** — только добавление.
- **Экспорт** (копия, каталог норм, слепки) — чтение, ничего не меняет.
- **Отправка слепков на сервер** — добавление на стороне сервера; локальное не меняется.
- **Смена профиля (logout)** — сбрасывает только сессию в памяти, не данные.

### Отдельно отмечено

- **Жёсткого удаления (delete) в интерфейсе нет вообще** — ни испытуемых, ни
  обследований, ни норм, ни методик. Это снижает риск потери, но и означает, что
  ошибочно созданную запись пока нельзя убрать (это функциональный пробел, не
  риск; если будем добавлять удаление — сразу по правилу: подтверждение + откат).

## Вывод

Оба реально опасных места (восстановление копии, импорт норм) закрыты и покрыты
тестами на «неправильный» путь. Два места среднего риска (правка нормы/методики
в месте) — обратимы или ограничены Owner-доступом; оставлены с рекомендацией
добавить предупреждение позже, отдельным решением владельца. Хард-делита нет.
