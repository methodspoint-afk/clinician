на GitHub Pages при пуше: **https://methodspoint-afk.github.io/clinician/**

**Сделано:**
- MVP: методики → замеры → подбор нормы (Gate → Score) → отчёт, локально (SQLite
  в браузере), без ФИО.
- Итерация 1 (обратная связь клинициста Степана): веса скоринга (свежесть 15→5,
  источник 10→20), возрастной Gate с жёсткими границами 16/60, формулировки
  «снижение/повышение показателя», 3 новые методики (воспроизведение чисел,
  сравнение понятий, пиктограммы), препараты/комментарий в карточке, подсветка
  обязательных полей, «ближайшие нормы» вместо пустоты, отчёт по блокам +
  динамика + экспорт .doc.
- Стартовая база норм: 7 норм из бэкапа Степана («Методичка Бехтерева», Шульте,
  «10 слов») досеваются при запуске.
- Обзор конкурентов и регуляторики: `clinician-os/competitor-review.md`.

Тесты: 46 юнит (vitest) + сквозной e2e (Playwright) — зелёные.

## Что дальше (согласованный порядок)

1. **[СЛЕДУЮЩЕЕ] Качественные методики** (задача #11, дорожка 1, локально).
   Карточки «исключение лишнего» и пиктограммы для набора базы ответов
   (слово → пояснение → квалификация специалиста), без подсказок системы.
2. **Решение владельца: сервер или нет** (задача #12). Общая база испытуемых,
   нормы по диагнозам, роли-администраторы, командный режим (платно),
   синхронизация — всё требует бэкенда + приватность/юридика. Обзор
   конкурентов подтверждает «нормы по диагнозам» как сильнейшую дифференциацию.
3. **Международный рынок**: английская локализация + «семейства методик»
   (TMT↔Шульте, RAVLT↔10 слов). После подтверждения ниши.
