> итерации 2 — качественные методики).

## Где мы сейчас

Ветка разработки: `claude/gifted-thompson-dsmsv5`. Демо публикуется автоматически
на GitHub Pages при пуше: **https://methodspoint-afk.github.io/clinician/**

**Сделано:**
- MVP: методики → замеры → подбор нормы (Gate → Score) → отчёт, локально (SQLite
  в браузере), без ФИО.
- Итерация 1 (обратная связь клинициста Степана): веса скоринга (свежесть 15→5,
  источник 10→20), возрастной Gate с жёсткими границами 16/60, формулировки
  «снижение/повышение показателя», 3 новые методики (воспроизведение чисел,
  сравнение понятий, пиктограммы), препараты/комментарий в карточке, подсветка
  обязательных полей, «ближайшие нормы» вместо пустоты, отчёт по блокам +
  динамика + экспорт .doc.
- Стартовая база норм: 7 норм из бэкапа Степана («Методичка Бехтерева», Шульте,
  «10 слов») досеваются при запуске.
- Обзор конкурентов и регуляторики: `clinician-os/competitor-review.md`.
- **Итерация 2 — качественные методики** (задача #11, готово): протокол-таблица
  без числовых норм. Две методики: «Исключение лишнего (4-й лишний)» (квалификация
  обобщения: существенный/функциональный/конкретно-ситуативный/латентный) и
  «Пиктограммы — протокол опосредования» (слово → образ → воспроизведённое слово).
  Система ничего не подсказывает — только копит базу (ТЗ, раздел 7). Модель:
  `MethodConfig.qualitative`, миграция v3 (`test_results.qualitative` JSON),
  `TestResult.qualitativeRows`, компонент `ui/QualitativeProtocol.tsx`; отражено
  в карточке и отчёте; нормы недоступны для качественных методик.

Тесты: 47 юнит (vitest) + сквозной e2e (Playwright, +шаг качественной пробы) — зелёные.
