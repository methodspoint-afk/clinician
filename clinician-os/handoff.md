## Что за проект

«Реестр патопсихологических методик» — веб-приложение (React + TypeScript + SQLite
в браузере через sql.js, локально, без ФИО испытуемых). Помогает клиническому
психологу: провести методику → сравнить результат с нормой → получить отчёт.
Ядро и отличие от конкурентов — **прозрачный подбор нормы Gate → Score**: сначала
отсев по популяции (возраст/образование/язык), потом балл качества нормы 0–100.
Приложение — поддержка решения специалиста, НЕ диагност (ключевой инвариант).

Заказчик — Степан (клинический психолог, тестирует MVP и даёт обратную связь).
Пользователь-владелец репозитория: methodspoint@gmail.com.

## Что сделано (хронология)

1. **MVP**: методики → замеры → подбор нормы (Gate→Score) → отчёт; локально, без ФИО.
2. **Публичное демо**: single-file HTML на GitHub Pages, защита от недоступного localStorage.
3. **Итерация 1** (обратная связь Степана): веса скоринга (свежесть D 15→5,
   источник E 10→20); возрастной Gate с жёсткими границами 16/60 и допуском ±10
   лет внутри взрослого диапазона (флаг `age_stretch`); формулировки вердиктов
   «снижение/повышение показателя относительно нормы условно здоровых»; 3 новые
   количественные методики (воспроизведение чисел, сравнение понятий, пиктограммы-
   количественная); поля «препараты» и «комментарий» в карточке (миграция v2);
   подсветка обязательных полей красным + обязательный год источника; «ближайшие
   нормы» вместо пустоты; отчёт по блокам (внимание/память/мышление) + динамика
   повторных обследований + экспорт .doc + комментарий специалиста по методике.
4. **Стартовая база норм**: 7 норм из бэкапа Степана («Методичка Бехтерева», Шульте
   ЭР/ПУ/ВР, «10 слов» p5/отсроченное) — `domain/seedNorms.ts`, досев при запуске.
   Тестового испытуемого из бэкапа НЕ добавляли (по прямой просьбе Степана).
5. **Обзор конкурентов и регуляторики**: `clinician-os/competitor-review.md`.
6. **Итерация 2** (качественные методики, задача #11): протокол-таблица без
   числовых норм. Методики «Исключение лишнего (4-й лишний)» (рубрикатор
   квалификации обобщения) и «Пиктограммы — протокол опосредования»
   (слово→образ→воспроизведённое слово). Система не подсказывает квалификацию —
   только копит базу (ТЗ раздел 7). Модель: `MethodConfig.qualitative`, миграция
   v3 (`test_results.qualitative`), `TestResult.qualitativeRows`, компонент
   `ui/QualitativeProtocol.tsx`. Нормы для качественных методик недоступны.

Итого методик: 9 (7 количественных + 2 качественные). Тесты: 47 юнит + e2e — зелёные.

## Предзапусковая надёжность (июль 2026)

**Безопасное восстановление резервной копии — готово.** `inspectBackup()` в
`database.ts` проверяет файл (таблицы приложения) и возвращает сводку; в
«Настройках» восстановление теперь в 2 шага с подтверждением («в файле N
испытуемых/обследований/норм/методик»), а перед заменой авто-скачивается
страховочная копия текущих данных. Чужой/битый файл отклоняется, данные не
трогаются. 63 юнит-теста + шаг в e2e (экспорт, отклонение чужого файла,
подтверждение по валидной копии) — зелёные.

## Статус С1 (июль 2026)

**Серверный MVP готов** (`server/`): Fastify + SQLite, эндпоинты /health,
/norms (публичный каталог), /norms/all + POST /norms + PATCH (админ),
/submissions (специалист), /aggregates (админ, ячейки n≥30 c mean/sd —
заготовка норм-кандидатов по диагнозам для С2). Bearer-токены без ПДн,
выпуск через `npm run token`. Форматы = файлам С0. 10 тестов API зелёные,
смоук запуска пройден. Деплой-инструкция: `server/README.md` (systemd,
Caddy/TLS, бэкапы sqlite .backup). SQLite вместо Postgres — осознанно для
варианта A; переезд на B меняет только `server/src/db.js`.

**HTTP-режим синхронизации — готов** (июль 2026): `app/src/domain/syncHttp.ts`
(fetch-клиент с инъекцией для тестов), настройки «адрес сервера + токен» в
settings, карточка «Сервер общей базы» в SyncScreen (сохранить настройки /
скачать нормы / отправить слепки), CORS на сервере (+2 теста). E2e гоняет
ЖИВОЙ сервер: поднимает server/ на 8788, выпускает admin-токен CLI, заносит
норму по API, UI скачивает её по HTTP — норма появляется в базе норм.
Тесты: app 60 + server 12 + e2e — зелёные.
