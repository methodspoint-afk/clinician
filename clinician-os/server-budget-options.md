# Бюджет сервера (этап С1) — три варианта

> **РЕШЕНИЕ ВЛАДЕЛЬЦА (июль 2026): подтверждён вариант A (~700–1300 ₽/мес).**
> Триггеры перехода на B зафиксированы ниже. Открыто: провайдер VPS, имя
> домена, список админов норм.

> Для решения владельца. Контекст: приложение локальное (offline-first),
> сервер — тонкий REST API + PostgreSQL (каталог норм, приём обезличенных
> слепков, админ-валидация). Нагрузка на пользователя — килобайты и редкие
> запросы, поэтому железо почти никогда не является узким местом; ключевая
> разница вариантов — **надёжность, обслуживание и готовность к коммерции**.
> Цены — ориентиры июля 2026 по российским провайдерам, проверить при закупке.

## Важная особенность нашей архитектуры (влияет на все варианты)

Если сервер упал — **специалисты продолжают работать**: вся повседневная
работа (обследования, нормы, отчёты) локальная; сервер нужен только в момент
синхронизации. Поэтому простой сервера у нас не «останавливает клинику», а
лишь откладывает обмен. Это позволяет честно начинать с дешёвого варианта.

## Вариант 0 — «Файлы» (текущий, 0 ₽/мес)

Продолжаем этап С0: обмен каталогом норм и слепками через файлы.

- **Возможности:** полный цикл «нормы → коллеги» и «слепки → к Степану»
  вручную; достаточно, пока пользователей ≈ Степан + пара коллег.
- **Ограничения:** ручная пересылка; нет накопления общей базы «само»;
  нет норм по диагнозам (С2).
- **Риски:** нулевые технические; продуктовый риск — «общая база» остаётся
  обещанием.
- **Когда достаточен:** до первых 5–10 внешних пользователей.

## Вариант A — «Старт» (~700–1300 ₽/мес) ← РЕКОМЕНДАЦИЯ для С1

Один VPS в РФ (2 vCPU, 2 ГБ RAM, 40 ГБ NVMe, ~500–900 ₽) + домен .ru
(~600–900 ₽/год ≈ 60–80 ₽/мес) + снапшоты/бэкапы (~150–300 ₽).
На VPS: Node/Fastify API + PostgreSQL + ежедневный дамп в отдельное хранилище.

- **Кому хватит:** до **~1000–2000 активных специалистов** по нашему профилю
  нагрузки (редкие синхронизации, лёгкие payload'ы); практически — весь
  горизонт С1–С2 и первые сотни пользователей.
- **Преимущества:** дёшево; данные в РФ; полный контроль; переезд на вариант B
  без переписывания кода (дамп Postgres → managed-база).
- **Ограничения:** одна машина = единая точка отказа (перезагрузка/сбой —
  сервис недоступен, но см. «особенность архитектуры»); обслуживание
  (обновления, безопасность, восстановление) — на нас; при аварии диска
  потеря данных с момента последнего бэкапа (смягчение: ежедневный дамп +
  WAL-архив каждые 15 мин — входит в те же деньги).
- **Риски:** админ-ошибки; нет staging-среды (тестируем аккуратно);
  SLA «как получится» — для бесплатной фазы приемлемо.
- **Что НЕ надо делать в этом варианте:** платные аккаунты с ПДн (email,
  оплата) — держимся инвайт-токенов без персональных данных.

## Вариант B — «Рост» (~3000–5500 ₽/мес)

Managed PostgreSQL у российского облака (авто-бэкапы, point-in-time recovery,
обновления — ~1500–3000 ₽) + контейнер/VM под API (~800–1500 ₽) + мониторинг
и staging (~500–1000 ₽).

- **Кому хватит:** тысячи–десятки тысяч специалистов; главное — **готовность
  к платному командному режиму**.
- **Преимущества:** база «не потеряется» (PITR, реплики по кнопке);
  обновления БД без нас; staging для безопасных релизов; мониторинг с
  алертами; проще пройти вопросы безопасности при разговоре с клиниками.
- **Ограничения:** ×4 к цене; лёгкая привязка к провайдеру.
- **Риски:** существенно ниже A; остаётся одна зона доступности.
- **Триггер перехода A → B:** любой из: >300–500 активных пользователей;
  запуск платного командного режима; появление ЛЮБЫХ персональных данных
  (email при регистрации команд) — тогда же policy по 152-ФЗ и юрист.

## Вариант C — «Коммерческий» (~10 000–25 000 ₽/мес)

Managed Postgres с репликой, 2 инстанса API за балансировщиком, объектное
хранилище бэкапов, мониторинг/алертинг, WAF, staging; при необходимости —
сегмент облака с аттестацией под 152-ФЗ.

- **Кому:** когда есть выручка, SLA-ожидания клиник и командный режим с
  реальными аккаунтами/оплатой.
- **Преимущества:** переживает отказ машины без простоя; аудит безопасности;
  комфортная почва для договоров с организациями.
- **Ограничения/риски:** цена; больше движущихся частей; нужен регулярный
  «взгляд админа» (или немного managed-услуг сверху).
- **Не раньше, чем:** подписка реально продаётся. Ставить это до выручки —
  сжигание денег.

## Рекомендация

1. **Подтвердить вариант A (~700–1300 ₽/мес) для С1** — он покрывает всё, что
   нам нужно на ближайшие месяцы, и ничем не блокирует рост: переезд на B —
   это перенос базы, не переписывание.
2. Заранее принять **триггеры перехода на B** (см. выше), чтобы решение потом
   не обсуждать заново.
3. Домен купить сразу (он же бренд); аккаунты хостинга/домена — на владельце
   проекта, не на разработчике.
4. Юрист по 152-ФЗ — не при варианте A (ПДн нет), а при переходе на B
   (команды/email) — это экономит деньги сейчас.
