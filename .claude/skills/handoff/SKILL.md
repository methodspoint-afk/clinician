# handoff — фиксация контекста проекта

Процедура (выполнять по порядку):

1. Обновить `clinician-os/handoff.md`:
   - что сделано в этой сессии (кратко, в хронологию «Что сделано»);
   - изменения статуса задач и плана;
   - новые решения владельца/клинициста со ссылками на документы;
   - актуализировать раздел «Дальше» и открытые вопросы.

2. Обновить `clinician-os/roadmap.md` в тон изменениям (статус, следующий шаг).

3. Если менялись зафиксированные ранее решения — обновить соответствующий
   доменный документ в `clinician-os/` (не только хендофф).

4. Закоммитить и запушить в ветку `claude/gifted-thompson-dsmsv5`:
   `git add -A && git commit -m "Хендофф: <короткое что зафиксировано>" && git push -u origin claude/gifted-thompson-dsmsv5`
   (сетевые сбои — до 4 ретраев с backoff).

5. В ответе пользователю — одной строкой сказать, что зафиксировано и где,
   и назвать следующий шаг.

Принцип: лучше обновлять хендофф часто (после каждого смыслового куска), чем
один раз «в конце» — сессия может оборваться внезапно.
